#western-union 

> [!info] 
> MT942 is the Interim Transaction Report (kind of like a “mini statement”).
It’s used when a customer (like Western Union or any corporate) wants intraday updates on their account, instead of waiting for end-of-day statements (MT940/950). 

```makefile
{1:F01BANKDEFFAXXX1234567890}{2:I942CORPUS33XXXXN}{4:
:20:123456789
:25:DE89370400440532013000
:28C:00001/001
:34F:EUR12345,67
:13D:2409301200+0100
:61:2409300930D2500,00NTRFNONREF//123456789
:86:/REMI/Invoice 8899 Payment
:61:2409301005C1000,00NTRFNONREF//987654321
:86:/REMI/Refund from Supplier XYZ
-}
```

*`:20:`* → Transaction Reference Number (unique ID).
*`:25:`* → Account Identification (IBAN or account number).
*`:28C:`* → Statement Number/Sequence.
*`:34F:`* → Balance for the Account
- Format:  `:34F:<Ccy><Balance>`
- Currency and balance reported at this time.
- Example: EUR12345,67 → EUR 12,345.67 available.

*`:13D:`*  → Date/Time Indicator
- Format: `:13D:YYMMDDHHMM+TZ`
- Indicates when this report snapshot was created.
- Example: 2409301200+0100 → 30 Sep 2024, 12:00 CET.

*`:61:`* → Statement Line (Transaction Entry)
- Format:`:61:<ValueDate><EntryDate><D/C><Amount><TransactionType><Reference>//<BankRef>`
- 2409300930 → Value date & entry date = 30 Sep 2024, 09:30
- D2500,00 → Debit of EUR 2,500.00
- NTRF → Transaction type = Transfer
- NONREF//123456789 → Bank reference numbers

*`:86:`* → Information to Account Owner (Narrative)
- Free text / structured data, e.g. payment purpose.
- Example: /REMI/Invoice 8899 Payment.
